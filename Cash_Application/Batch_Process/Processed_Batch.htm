Processed Batches

The Processed Batches page provides a consolidated view of all factoring-processed cheques within a batch. Both successfully processed batches and failed batches are displayed here, giving users full visibility into the processing outcome of each batch.

An important control on this page is the restriction on the Undo Batch operation. To maintain data integrity and prevent inconsistencies, the undo operation is not permitted for processed batches. Validation rules are enforced by the system to detect such cases and block the undo action automatically.

Batches that failed during processing are visible on this page alongside successful batches.

The  field indicates the status of any batch that did not process successfully, allowing users to identify and investigate issues without navigating away from the page.

Fields & Descriptions:

Fields

Descriptions

Batch No

The unique identifier assigned to each batch entry at the time of Cash Batch creation.

Use this number to locate, track, and manage individual batches within the application.

Ref Batch No

 The Reference Batch Number associated with a batch entry, assigned during Cash Batch creation. 

Batch Date

The date on which the batch was created or processed.

This field supports chronological tracking, batch management, and audit trail purposes.

Entered during Cash Batch creation.

Batch Amount

The total cumulative value of all payments or cheques contained within the batch.

Used for financial reporting, reconciliation, and auditing.

Check Totals

The total cumulative value of all cheques included in the batch.

Used to reconcile and verify the financial details of the batch.

Failure

 Displays the failure status for processed batches that did not complete successfully.

Total Processed Batches

 A count of all processed batches displayed on the page.
