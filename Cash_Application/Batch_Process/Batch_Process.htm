Batch Process

The Ineligible page lists all invoices marked as ineligible.

When you hover over a value in the Invoice Status column, a tooltip shows the specific reason the invoice failed validation.

The Batch Process page provides a consolidated view of all batches used to process payments and repayments within the Factoring module.

Each batch may contain one or more payments or checks, and serves as the primary control point for validating and processing batch-level financial transactions.

The core objective of the Batch Process page is to ensure that each batch reaches a Balanced status before it is eligible for payment processing.

Fields & Descriptions:

Fields

Descriptions

Batch No

This field specifies the unique Batch Number assigned to each batch entry.

The Batch Number is system-generated and serves as the primary identifier for distinguishing and tracking individual batches within the application.

It enables users to quickly locate, manage, and review specific batches, especially when working with multiple batches.
Ref Batch No

This field specifies the Reference Batch Number associated with the batch entry.

It may be used to link the batch to an external system or to reference another batch within the application.
Batch Date

This field specifies the date associated with the batch entry.

The Batch Date represents the day on which the batch was created or processed.

This date is critical for organizing batches chronologically and supports batch management, reporting, and audit trail requirements.

Batch Amount

This field specifies the total amount associated with the batch entry.

It represents the cumulative monetary value of all payments or checks included in the batch.

The Batch Amount is a key financial indicator and is used for reconciliation, financial reporting, and auditing purposes.

Check Total

This field specifies the total amount of all checks associated with the batch entry.

It represents the combined monetary value of the checks included in the batch.

The Check Total is used to verify batch accuracy and to ensure alignment with the Batch Amount during validation.

Status

This field specifies the current validation and processing status of the batch.

The following statuses may be displayed:

Balanced
Indicates that the Batch Amount, Check Totals, and Applied Totals are equal.
A balanced batch reflects accurate and consistent financial data and is required for payment processing.

Not Balanced
Indicates that the Batch Amount, Check Totals, and Applied Totals do not match, resulting in a financial discrepancy that must be resolved before processing.

No Checks Added
Indicates that the batch has been created but does not contain any checks.
This status typically occurs when a batch is created before payment details are entered.

Account(s) Missing
Indicates that debtor information has not been selected during invoice-level payment processing.
Debtor association is required to correctly apply payments to invoices.

Check No Missing
Indicates that one or more checks in the batch do not have a check number entered.
Check numbers are required for accurate tracking and documentation.

Balanced / In Progress
Indicates that the batch is financially balanced and that batch processing is currently underway.
This is a transitional status between initiation and completion of processing.

Balanced / Check Distribution is Out of Balance
Indicates that the Batch Amount and Check Totals are equal, but the distributed amount is less than the check amount.
This reflects a discrepancy at the check distribution level.

Not Balanced / Check Distribution is Out of Balance
Indicates that the Batch Amount and Check Totals are not equal, and the distributed amount is less than the check amount.
Both batch-level and distribution-level discrepancies exist and must be resolved.
