Ungrouped Checks

The primary use for the Ungrouped Checks page comes after the Deposit Uploads.

After uploading a file through Deposit Uploads, one or more batches are created in the Batch Process screen.

While reviewing checks within a batch, users may encounter complexities that prevent immediate processing. In such cases, users may need to process the current batch while excluding specific checks.

Common Reasons for Excluding Checks:

  • Client/Borrower is unidentifiable.

  • Required data is missing.

  • Internal approval is required before processing.

The goal for the selected checks is to:

  • Retain certain checks in the current batch.

  • Move complex checks out of the batch for later review.

  • Process the remaining checks in the batch.

    You can also exclude checks from a batch on the check entry page. Here is how:

    1. Select the check in scope.

    2. Click the  button to send the selected check to the Ungrouped Checks, where the selected check is excluded from processing.

Batch No

This field specifies the unique Batch Number assigned to each batch entry.

The Batch Number is system-generated and serves as the primary identifier for distinguishing and tracking individual batches within the application.

It enables users to quickly locate, manage, and review specific batches, especially when working with multiple batches.
