Factoring Charge Codes

Factoring Charge codes are used to record transactions at each stage of an invoice journey.

The system allows users to add and modify charge codes based on configuration requirements. This helps manage all costs and expenses related to factoring, making it easier to monitor, report, and control these financial transactions.

Fields & Descriptions:

Fields

Descriptions
Charge Code

Displays the names of all charge codes, including those created by users and seed data preloaded into the system.
NAT Sign

Displays the natural sign of the charge code, either positive or negative.

  • Advances, fees, and interest carry a positive sign.
  • Collections, repayments, and reversals carry a negative sign.

Sequence 

Displays the order in which charge codes are arranged.

Users can set and modify the sequence at any time after creation.
Frequency 

Displays how often the charge code is applied.

Available options include: Annually, Business Daily, Daily, Manual, Monthly, Quarterly, Weekly, and One Time.
Charge Type 

Displays the basis on which the charge code value is calculated.

There are four charge types.

  • Absolute calculates the charge based on the value configured in the Charge Code page.
  • Advance calculates the charge based on the advance amount.
  • Invoice calculates the charge based on the invoice amount.
  • Loan calculates the charge based on the loan amount.

Value 

Displays and allows the user to enter the value for the charge code. Defaults to 0.00.

The value entered here is used by the system to calculate charges.
In Balance

Displays a checkbox for each charge code. Enabled by default.

The column is editable, but changes are not saved.

Auto New Client

Displays a checkbox for each charge code. Enabled by default.

The column is editable, but changes are not saved.
