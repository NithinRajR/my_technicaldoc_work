Manual Entry

The Manual Entry page lets you manually create and process invoices. Any invoice you enter here will appear on the page after processing.

Use the Manual Entry grid to add invoices for a specific client–debtor relationship.

You can:

  • Click the  button to process the invoices later, or
  • Click the   button to process invoices immediately for further workflow actions.

You can add and process up to 100 invoices simultaneously in the Manual Entry grid.

    To manually add invoices, follow the steps below.

    1. Select the client from the dropdown

    • (Optional)If you have an invoice file or image to attach, upload it using the Upload File field.

    2. In the Fee Name field, select the fee name from the drop-down list.

    3. Enter the details like

    • Invoice Number
    • Account Number
    • Account Name

    Note: CYNC automatically creates new account debtors when you enter the Account Number and Name. 

    4. Enter the optional fields like

    • PO Number
    • File Ref. No.

    5. Enter the Invoice Amount and Invoice Date.

    6. Select the Payment Terms.

    Note: To add more options, go to Factoring → Maintenance → Term Codes.

    7. CYNC automatically calculates the Due Date based on the Invoice Date and Payment Terms.

    8. Select the Invoice Status.

    9. Click the  button to insert more rows if needed.

    10. Click the  button to process the entries. CYNC processes the entries, generates a scheduler number, and moves the invoices to the New Invoices page.

    11. Click the  icon to delete an invoice entry.

Fields & Descriptions:

Fields

Descriptions

Invoice Number

This field captures the invoice number. Each invoice number must be unique.

If the same invoice number appears in multiple rows, the system displays:

“Duplicate invoice(s) have not been entered. Please check.”

You cannot process invoices until the duplicate is corrected.

Refer to the screenshot:
Account Number

This field displays the debtor’s account number as maintained in the Account Debtor page.

  • If you leave this field blank and click Save or Process Invoices, the system uses the Account Name value as the Account No.

  • If the entered Account No already exists, the system automatically fills the Account Name field.

    • Example: If Account No Sys02 belongs to Systems Inc.+, selecting Sys02 auto-fills Systems Inc.

  • If the Account No and Account Name do not exist in CYNC, the system creates a new account on the Account Debtor page after you click the  button and the invoices are successfully processed.

Limit: Maximum 50 characters.
Account Name

This field captures the debtor’s account name and is mandatory.

  • You can select an existing account name or enter a new one.

  • If the entered Account Name already exists, the system auto-fills the associated Account No.

    • Example: Selecting Systems Inc auto-fills Sys02.

  • If both Account No and Account Name are new, CYNC creates a new account record after successful invoice processing.

Limit: Maximum 50 characters.

PO Number

An optional field to capture the purchase order number.

If you exceed 20 characters and click Process Invoices, the system displays:

“PO Number should not be more than 20 characters.”

File Ref. Number 

An optional field to capture the file reference number.

Invoice Amount

Mandatory field that captures the invoice amount.

  • If you leave it blank, the system displays:
    “Invalid details. Please update the fields marked in red.”
    The field is highlighted in red.

  • Values less than 0.01 cannot be processed.
    If entered, the system shows:
    “Please enter an Amount greater than or equal to 0.01 in order to proceed.”

  • If the amount has more than three decimal places, the system rounds it to two decimal places after processing and displays the rounded value on the New Invoices page.

Invoice Date

Use the date picker to select the invoice date. This is a mandatory field.

Payment Terms

Select payment terms from the list of values.

Only active term codes from Factoring → Maintenance → Term Codes appear here.

The Due Date is automatically calculated using:

Invoice Date + Number of days in the selected term code.

Due Date

 System-generated value based on the Invoice Date and selected Payment Term.

Invoice Status

By default, the only value available is New.

Additional statuses can be configured in: 

Menu → Factoring → Maintenance → Invoice Status

Newly added statuses appear in this list of values

Fee Name

The Fee Name list displays all fee names defined on the Factoring Fee Setup page.

  • You must assign a fee name to every invoice entered in the Manual Entry grid.

  • For clients using a Tier-Based Fee, the Fee Name list is prefilled with the single available tier-based fee.

  • For clients using a Per Day Rate, multiple fees may be configured. In this case, you must select the appropriate fee name from the Fee Name list.

Delete

Use the Delete icon to remove a specific invoice row.

Total Amount

Displays the sum of all invoice amounts in the Manual Entry grid, rounded to two decimal places.
