  • Verification Batch DetailsVerification Batch Details

    When you select an invoice and click the  button, the system generates a Verification Number. This number appears as a hyperlink in the Verification No column.

    The subsequent verification steps depend on the Verification Mode configured for the debtor.

    Visit the Account Debtor page to know more about the Verification Mode.

    Email-Based Verification

    If Email is selected as the Verification Mode:

    The invoices are automatically sent to the debtor for approval.

    a. Approved Invoices

    • If the debtor approves the invoice, it appears on the Verified page.

    • The invoice status displays as “Approved”.

    b. Declined Invoices

    • If the debtor declines the invoice, it appears on the Ineligible page.

    • The invoice status displays as “Declined”.

    Manual Verification 

    If any other verification mode (e.g., phone/call) is selected, the CYNC user must manually complete the verification. Follow the steps below:

    1. Open the Verification Batch

    Click the hyperlinked Verification No on the Pre-Verification page.

    The Verification Batch Detail pop-up opens.

    In the Verification Method dropdown, select Call.

    After selecting Call, click the  button to open the Verification URL, where invoice-level actions can be taken.

    Choose one of the following options:

Payment Terms

Column in ‘Pre-Verification’ page to display Payment Terms associated with each invoice.

Payment terms can be set in the “Terms Code” page for the respective client. Only the active term codes from the Terms Code page can be assigned to an invoice, and the list of active term codes is only available in the Payment Terms column LOV.

Users can select any value from the Payment Terms column LOV (column header) to search invoices associated with the selected payment term code. Values in the Payment Term column LOV are fetched from the “Menu → Factoring → Maintenance → Term Codes” page.

Due Date

The due date is computed based on the Invoice date and the number of days in the payment terms for an invoice.

Example:

Invoice Date: 01/05/2021

Payment Terms = Net 30 & the number of days in these payment terms is 30

Due Date = Invoice Date + Number of days in selected Payment Term

Due Date = 01/05/2021 + 30 = 01/31/2021

Fee Name

A column in the ‘Pre-Verification’ page to display the fee name assigned to an invoice.

Fee Name(s) for a factoring fee setup can be set in the Factoring Fee Setup page for the respective client.

The user can select any value from the “Fee Name” LOV (column header) to search invoices associated with the selected fee name.

Values in the Fee Name LOV are fetched from the “Menu → Client Maintenance → Factoring Fee Setup” page.

Note: Fee Name LOV will have only one fee name value in case a client is using the Tier-Based fee setup, as currently, only one fee can be set up for the tier-based fee setup.

For per day rate fee setup, multiple fees can be set, and names can be assigned to each fee setup. In case a client is using a per-rate fee setup with multiple fee setups in the factoring fee setup page, all the active fee names should be available in the Fee name column header LOV.

Average days to pay

Average Days to Pay (ADP) indicates how many days, on average, a debtor takes to pay their invoices. It helps the user evaluate the debtor’s payment behavior.

Higher ADP → Higher risk (debtor takes longer to pay)

Lower ADP → Lower risk (debtor pays faster)

Computation Logic:

ADP is calculated by taking the average of the Age Days for all payments made by the debtor.

Age Days Formula:
Age Days = Payment Date – Invoice Date

Payment Date is the date when the invoice is fully paid (invoice balance becomes 0).
