Cash Receipt

The Cash Receipt page displays all payment transactions posted in CYNC. This includes payments applied through Cash Application to both factored and non-factored invoices, as well as chargeback and recourse transactions.

The following payment transaction types are shown on the Cash Receipt page:

  • PMT – Payment
  • PP – Pre-Payment
  • SP – Short Payment
  • OP – Over Payment
  • NF / NF-UAC – Non-Factored / Non-Factored Unapplied Cash
  • CB – Chargeback (partial and complete)
  • RT – Recourse Transaction (for Recourse clients)

Here is a quick look at the application page:

Fields & Descriptions:

Fields

Descriptions
Account Number

Displays the debtor account number associated with the transaction.
Invoice Number

Displays the invoice number linked to the transaction.

Invoice Date

Displays the invoice date in MM/DD/YYYY format.
Due Date

Displays the due date of the invoice.
Funding Batch

Displays the funding batch number associated with the invoice, if applicable.
Payment Date

Displays the date on which the payment was posted, in MM/DD/YYYY format.

When a past-dated payment is processed through Cash Application, the corresponding transaction is added to the Cash Receipt page using the entered payment date.

  • Past-dated payments processed through Cash Application appear based on the entered payment date.

  • Records are sorted by Payment Date in descending order by default, with the most recent date shown first.
Invoice Amount

Displays the original invoice amount.
Payment Amount

Displays the payment amount applied through the Batch Process Screen in Cash Application.

  • For Chargeback (CB) and Recourse (RT) transactions, this value is always 0.00.
Applied Amount

Displays the total amount applied to the invoice.

Applied Amount is calculated as: Applied Amount = Payment Amount + Adjustment Amount

Examples:

  • PMT / PP / OP / NF / NF-UAC → Applied Amount equals Payment Amount

  • SP (Short Payment) → Applied Amount = Payment Amount + Adjustment Amount

  • CB / RT → Applied Amount equals Adjustment Amount
Adjustment Amount

Displays the adjustment portion of a transaction.

Adjustment Amount is calculated as: Adjustment Amount = Applied Amount − Payment Amount

  • Displayed for SP, CB, and RT transactions

  • Deducted from the client’s Cash Reserve Balance
Adjustment Reason

Displays the adjustment reason selected during processing.

  • Applicable only for Short Payment (SP) transactions
Trans Type

Displays the transaction type:

  • PMT – Payment

  • PP – Partial Payment

  • SP – Short Payment

  • OP – Over Payment

  • NF / NF-UAC – Non-Factored / Non-Factored Unapplied Cash

  • CB – Chargeback (partial or complete)

  • RT – Recourse Transaction
Source

Displays the source of the payment.

  • Source values are configured in Cash Receipt Source Code with target codes Repayment or Collection

  • Only configured source codes are available during payment processing
Check No

Displays the check number used for payment processing.

  • Blank for Chargeback (CB) transactions

  • Blank for Recourse (RT) transactions
Batch No

Displays the batch number generated during payment processing.

  • Populated only for payments processed through Cash Application

  • Blank for Chargeback and Recourse transactions
Totals

Displays grand totals for the following columns:

  • Invoice Amount

  • Payment Amount

  • Applied Amount

  • Adjustment Amount
