New Invoices

The New Invoices page displays all invoices you processed through Manual Entry, Factoring File Uploads, or Import QuickBooks Invoices.

It also shows invoices you accepted from the Duplicate Invoices screen.

    The  button sends the selected invoices to the next workflow stage.

    During processing, the system applies the client’s Invoice Validation Template to check whether the invoices meet all required parameters.

    If the invoices fail any validation, the system routes them to one of the following screens:

    • Ineligible
    • Pre-verification

    Note: If the parameter is set to “No Invoice Verification Required,” the system skips verification when you click the Process button and sends the invoices directly to the Verified screen.

Fields & Descriptions:

Fields

Descriptions

Invoice Number

The system displays the invoice number for each record processed through Manual Entry, Factoring File Uploads, or QuickBooks imports.
Account Number

This column displays the debtor’s account number for each invoice listed on the New Invoices page.
Account Name

This column displays the debtor’s account name for each invoice.

The value is a hyperlink. When you click it, a pop-up opens showing the following details:

  1. Invoice Details

Refer to the screenshot:

  1. Account Parameters

  2. Comments

  3. Invoice Documents

Invoice Date

This column displays the invoice date in MM/DD/YYYY format.

Payment Terms

This column displays the payment terms assigned to each invoice. These terms are created in the Term Codes module.

Due Date

This column displays the computed due date in MM/DD/YYYY format.

The system calculates the due date using the invoice date and the number of days defined in the selected payment term.

Example:

  • Invoice Date: 01/05/2021

  • Payment Terms: Net 30 (30 days)

  • Due Date = 01/05/2021 + 30 days = 01/31/2021
Invoice Amount

This column displays the invoice amount.
PO Number

This column displays the purchase order number associated with an invoice. If no PO is provided, the field remains blank.

File Name

This column displays the file name used during processing on the Factoring File Uploads page and during QuickBooks imports.

For invoices entered through Manual Entry, this field remains blank.
File Ref. Number 

This column displays the reference number provided either in the Manual Entry page or mapped as the Batch Reference Number during file uploads.

If no File Ref. No. is provided, the field remains blank.
Schedule Number

The system automatically generates the schedule number.

The value is a hyperlink. Clicking it opens the schedule report in HTML format in a new browser tab.

Refer to the screenshot:

Invoice Status

This column displays the invoice status carried over from the Manual Entry screen.

Fee Name

This column displays the fee name assigned to each invoice.

You can filter invoices by selecting a value from the Fee Name column header.

Notes:

  • For Tier-Based fee setups, the LOV displays only one value because only one fee can be configured.

  • For Per-Day Rate setups, multiple fees can be configured, each with its own name. If multiple fees exist, all should appear in the Fee Name LOV.
