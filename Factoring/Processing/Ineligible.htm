Ineligible

The Ineligible page lists all invoices marked as ineligible.

When you hover over a value in the Invoice StatusInvoice Status column, a tooltip shows the specific reason the invoice failed validation.


Invoices move to this page from the New Invoices page whenever a client-specific validation rule fails.

You can override a failed validation by clicking the  button, which marks the invoice as eligible. After an invoice is marked ineligible, the system routes it to either the Pre-Verification page or the Verified page, depending on the client’s validation rule setup.

An invoice is marked as Ineligible when it fails one or more validation parameters configured in the Invoice Validation Template.

  • Recalculate InvoicesRecalculate Invoices

    The  button appears on the Ineligible page and allows you to re-evaluate invoice eligibility after updates to the Invoice Validation Template.

    When to Use

    Use this option when validation parameters have changed, and the eligibility of invoices already listed on the Ineligible page needs to be recalculated.

    How It Works

    • Recalculate Selected Invoices

    Select one or more invoices using the checkboxes in the grid, then click the  button. The system recalculates eligibility only for the selected invoices.

    • Recalculate All Invoices

    If you click the  button without selecting any invoices, the system recalculates eligibility for all invoices displayed on the Ineligible page.

  • Override InvoicesOverride Invoices

    The  button on the Ineligible page lets you mark selected invoices as eligible for further processing.

    Once overridden, the invoice moves to either the Pre-Verification page (when verification is required) or directly to the Verified page (when no verification is needed).

    Note: The  button is available only when the Override checkbox is enabled in the Invoice Validation TemplateInvoice Validation Template.

    1. Select the invoice you want to override by clicking the checkbox next to it.

Fields & Descriptions:

Fields

Descriptions

Invoice Number

This column displays the invoice number.

If an invoice is declined and appears on the Ineligible page, a red warning icon (triangle) is shown next to the invoice number.
Account Number

Displays the debtor’s account number for each invoice listed on the Ineligible page.
Account Name

Displays the account name of each debtor.

Each account name is a hyperlink,hyperlink, and clicking it opens a pop-up with the following links:

  • Invoice Details

Refer to the screenshot:

  • Account Parameters

  • Comments

  • Invoice Documents

Invoice Date

Shows the date of the invoice in MM/DD/YYYY format.

Payment Terms

Displays the payment terms assigned to the invoice.

  • Payment terms are configured on the Terms Code page.

  • Only active term codes appear in the LOV.

  • Users can select only from active codes.

Due Date

Shows the invoice due date in MM/DD/YYYY format.

The due date is calculated as:

Due Date = Invoice Date + Number of Days in Payment Terms

Example:
Invoice Date: 01/05/2021
Payment Terms: Net 30
Due Date: 01/31/2021
Invoice Amount

Displays the amount of the invoice.
PO Number

Displays the Purchase Order number associated with the invoice.

If no PO is provided, the field remains blank.

Invoice Status

Displays the status of the invoice.

  • Users can filter invoices using the Invoice Status LOV, which fetches values from the Invoice Status page.

  • Hovering over any invoice status displays a tooltiptooltip containing the validation rule(s) that failed.

  • If multiple rules failed, all are shown in the tooltip.

Fee Name

Displays the fee names configured for the client.

  • For Tier-Based Fee Setup: Only one fee name appears in the LOV.

  • For Per Day Rate Fee Setup: Multiple fee names may exist; all configured fee names appear in the LOV.

Schedule Number

Displays the schedule number for each invoice.

Each value is a hyperlink that opens the Schedule No report in HTML format in a new browser tab.

Comments

Users can enter individual comments for each selected invoice.

If invoices are overridden with different comments, each comment must be entered in its respective row.

Totals

Located at the bottom of the page, this field displays the total of all Invoice Amount values.
