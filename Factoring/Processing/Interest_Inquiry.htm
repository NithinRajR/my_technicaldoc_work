Interest Inquiry

The Interest Inquiry page provides a read-only view of interest charges generated by the Factoring application. It displays all transaction records associated with interest charges for the selected client(s).

This page is view-only, with the exception of the  button. Only records belonging to the selected client are displayed.

Users can filter records by specifying a From Date and To Date.

By default, the page displays transactions from the most recent three-month period.

The default To Date is the current date, and the default From Date is three months prior to the current date. The To Date may be backdated.

Fields & Descriptions:

Fields

Descriptions
From Date and To Date

Use the From Date and To Date fields, with the date picker, to define the transaction date range.

The system displays transaction records in the Interest Inquiry page based on the selected range.

Dates use the MM/DD/YYYY format. Users may select dates from the date picker or manually enter dates in MM/DD/YYYY format.

By default, the page displays transactions from the last three months. The system sets the To Date to the current date and the From Date to three months prior (current date minus three months).

Users may backdate the To Date. 
Transaction Type

Displays the transaction type for the selected client and is as follows.

Interest Charge

  • Displays interest charged to the client based on the Interest Setup configuration. The value appears as a hyperlink. Clicking the link opens a details pop-up.

Interest Charge Adj

  • Displays interest adjustment entries created through Adjustment Entry.
  • Users may backdate adjustments from the 16th of the previous month through the last day of the current month. The system recalculates interest from the day following the adjustment date through the current date. The value appears as a hyperlink.

Funding

  • Displays funding transactions processed through funding batches. These entries mirror funding records shown on the Funded Invoices page. The value appears as a hyperlink.

Funding Adj

  • Displays funding adjustment entries created through Adjustment Entry. Users may backdate funding adjustments from the 16th of the previous month through the last day of the current month. The system recalculates interest from the day following the adjustment date through the current date. The value appears as a hyperlink.

Payment

  • Displays payment transactions processed through Cash Batch Processing, including PMT, PP, OP, and SP. NF and NF-UAC transactions do not affect interest and do not appear on this page. The value appears as a hyperlink.

Chargeback

  • Displays full and partial chargeback transactions processed through the Funded Invoices page. The value appears as a hyperlink.

Reserve Posting

  • Displays accrued interest posted as a reserve deduction on the last day of the month. The system deducts this amount from the client’s reserve balance. The value appears as a hyperlink.

Transaction Date

Displays the transaction date for each entry in MM/DD/YYYY format.

The system sorts transactions in descending order by default, with the most recent transaction displayed first. Users may sort the column in ascending or descending order.
Transaction Amount

Displays the transaction amount for each transaction type.
Opening Balance

Displays the interest-chargeable opening balance.

The system sets the opening balance equal to the prior transaction’s closing balance.
Funding

Displays funding amounts associated with the transaction.
Interest Charge

Displays the interest charged for the day based on the opening balance.
Payments / Chargeback

Displays all payment and chargeback transactions.

Payment types include PMT, PP, SP, and OP. Chargebacks include both full and partial chargebacks.
Reserve Deductions

Displays reserve deductions posted on the last day of the month.

  • Reserve Deductions = Interest Charge + Interest Charge Adjustments

The system posts this amount as a Reserve Posting and deducts it from the client’s reserve balance.
Closing Balance

Displays the interest-chargeable closing balance.

If Interest Posted To = In Balance
Closing Balance = Opening Balance + Interest Charge

If Interest Posted To = Accrue To Balance
Closing Balance = Opening Balance
Accrued Interest

Displays accrued interest for the client.

  • Daily Accrued Interest = Interest Charge + Interest Charge Adjustment

The system sums accrued interest from the first through the last day of the month and posts the total as a reserve deduction.
Cumulative Interest Amount

Displays the running total of interest charged.

  • Cumulative Interest = Prior Day Cumulative Interest + Current Day Interest Charge

The system carries this value forward across months.
Cumulative Payments / Chargeback

Displays the running total of payments and chargebacks for the current month.

The system resets this value at the start of each month.
Cumulative Reserve Deductions

Displays cumulative reserve deductions through the end of the current month.

  • Cumulative Reserve Deductions = Prior Month Reserve Deductions + Current Month Reserve Deductions
Cumulative Funding

Displays the total funding amount for the client.

  • Cumulative Funding = Prior Cumulative Funding + Current Day Funding

The system carries this value forward across months.
Cumulative Funding Balance

Displays the cumulative funding balance.

  • Cumulative Funding Balance = Prior Balance + Current Funding − Current Payments
Interest %

Displays the interest rate used to calculate interest charges.

  • Interest % = Interest Rate Code + Adjustment Rate %

For daily interest frequency, the system applies the prevailing rate for each day.
For monthly interest frequency, the system applies the latest rate on the last day of the month.
Hide Interest Holiday and No Transaction

Select this checkbox to hide zero-transaction records.

The checkbox is selected by default. Clearing the checkbox displays zero-interest and no-transaction records.
